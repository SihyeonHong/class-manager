# 기획

## 의도

브라우저 알림은 정상적으로 발송되나 소리가 출력되지 않는 문제를 해결하고자 한다.

## 문제 상황 분석

`new Notification(...)`을 통해 데스크톱 알림을 발송하고 있으나 소리가 출력되지 않음.
`useStudentManager.ts` 270줄 전후 코드 참고.

### 확인된 환경

- 크롬 브라우저
- 브라우저 설정: 해당 사이트의 알림 허용 받음, 소리 재생 허용.
- 사용자 상호작용(User Interaction)은 이미 보장.

> c.f. 사용자 상호작용(User Interaction):
> 브라우저는 사용자가 페이지와 최소 1회 이상 상호작용(클릭, 키 입력 등)을 하지 않은 상태에서 발출되는 audio.play()를 차단하는 정책이 있다. 사이트 진입 후 사용자가 알림 권한을 승인하거나, 서비스 내 버튼을 클릭하는 시점 이후부터 소리 재생이 정상 작동한다.
> 우리 프로젝트의 경우, 반드시 키 입력을 통한 학생 등록 + 사용자의 사이트 알림 권한 승인 절차가 선행되므로 이 부분은 고려하지 않아도 된다.

### 추정 원인

##### 기본 소리 출력 방식

`new Notification()`으로 생성된 알림은 브라우저 자체 음향이 아닌, 운영체제(OS)의 기본 웹 알림 음향 설정에 의존.

크롬 브라우저 및 현대 OS 환경에서는 Web Notification 사용 시 별도의 사운드 출력을 기본적으로 보장하지 않거나, OS 방해 금지/알림 설정에 따라 소리가 생략되는 경우가 많다.

#### silent 및 sound 속성의 제약

`NotificationOptions` 표준에는 `silent: false` 옵션이나 일부 환경의 `sound` 속성이 존재하지만, 크롬을 포함한 대부분의 주요 데스크톱 브라우저에서는 이를 지원하지 않거나 무시.

### 추정 해결 방안

알림 트리거 시점에 오디오 파일을 직접 재생.
HTMLAudioElement 또는 Web Audio API 등 활용할 것으로 추정.

# 과정

## 1. 탐색

1. HTML5 `Audio` 객체 사용하기로 결정.
2. 오디오 파일 구해야 함.

### 근거: 필요한 기능이 아주 단순함

- 짧은 알림음 하나를 1회 재생
- 여러 학생의 알림이 동시에(또는 1초 이내의 간격으로) 울릴 경우, 먼저 울리던 알림음을 도중에 끊고 바로 다음 알림음을 가로채듯이 재생

이는 오디오 사용 관련해서 제일 단순한 기능.
여러 오디오 겹치게 재생, 반복 재생, 미세한 타이밍 제어 등이 필요하지 않다.
향후 기능 확장 가능성도 낮다.
따라서 구현이 가장 간단한 HTML5 `Audio` 객체만으로 충분하다.

- 가공 없이 단독 파일 재판매 재배포 금지:, HTML 'Audio' 객체를 통해 웹서비스의 알림음/효과음으로 내장하여 제공하는 방식은 '단독 파일 재배포'가 아닌 '서비스 구성 요소로서의 결합 활용'에 해당하므로 정책상 안전하게 사용 가능.
