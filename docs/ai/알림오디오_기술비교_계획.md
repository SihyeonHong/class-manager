# 웹 알림과 함께 오디오 재생 방식 탐색 계획

## 1. 목적

브라우저 알림과 함께 재생할 알림음의 방식과, 실제로 사용할 오디오 자원을 어떻게 확보할지에 대해 정리한다.

본 문서는 코드 구현 단계가 아니라, 기술적 검토와 선택 기준을 정리한 문서이다.

---

## 2. 핵심 판단

현재 요구사항은 “짧고 단순한 알림음 재생”에 가깝다. 다만 실제 사용 환경에서는 여러 학생의 알림이 거의 동시에(또는 1초 이내의 짧은 간격으로) 발생하는 경우가 빈번할 수 있다.

이 경우 다음과 같은 동작 방식이 가장 적절할 것으로 보인다.

- 새 알림이 들어오면 이전에 재생 중이던 사운드를 즉시 중단한다.
- 바로 다음 알림음을 재생하여, 이전 사운드가 끝날 때까지 기다리는 방식이 아니라 “최신 알림음이 우선”하게 처리한다.

즉, 짧은 사운드 재생이라도 “동시성”을 고려한 제어가 중요하다.

---

## 3. 기술 방식 비교 요약

### HTML5 `Audio` 객체 (`new Audio()`)

- 장점
  - 구현이 가장 단순하고 진입 장벽이 낮다.
  - 짧은 알림음 재생에는 충분히 적합하다.
  - `pause()`, `currentTime = 0`, `play()` 조합으로 “새 알림이 들어오면 이전 사운드를 끊고 바로 다음 사운드를 재생”하는 흐름을 구현하기 쉽다.
  - 브라우저 지원 범위가 넓고, 웹 환경에서 가장 보편적이다.

- 단점
  - 복잡한 오디오 처리나 미세한 타이밍 제어는 어렵다.
  - 여러 사운드의 겹침, 믹싱, 음량/이펙트 제어는 제한적이다.

### Web Audio API (`AudioContext`)

- 장점
  - 타이밍, 볼륨, 효과음 처리, 음향 제어가 매우 유연하다.
  - 향후 사운드 기능을 확장할 가능성이 있다.

- 단점
  - 구현 복잡도가 높고, 코드량과 관리 포인트가 늘어난다.
  - 현재 요구사항처럼 단순한 알림음 재생만을 위해서는 다소 과한 방식일 수 있다.

### 외부 라이브러리 (예: `howler.js`)

- 장점
  - 여러 오디오 자원을 관리하는 데 편리하다.
  - 브라우저 간 차이를 줄이거나, 반복 재생 같은 기능을 다루는 데 유리하다.

- 단점
  - 의존성 추가와 번들 크기 증가가 있다.
  - 현재처럼 짧은 알림음 재생만 필요할 때는 오버엔지니어링일 수 있다.

---

## 4. 프로젝트 관점의 권장 방향

현재 프로젝트의 목적에 비추어 보면, 초기 구현은 HTML5 `Audio` 객체가 가장 현실적이다.

특히 다음과 같은 방식이 적합하다.

- 하나의 오디오 인스턴스를 재사용한다.
- 새 알림이 발생하면 기존 재생을 중단하고, 사운드 위치를 초기화한 뒤 새 파일을 재생한다.
- 이 방식은 짧은 알림음에 충분히 적합하고, 여러 알림이 연속으로 발생하는 상황에서도 비교적 자연스럽게 대응할 수 있다.

즉, “가장 최신 알림이 가장 먼저 들리도록” 처리하는 방식이 핵심이다.

---

## 5. 오디오 자원 확보 방안

현재 프로젝트에는 오디오 파일이 없으므로, 사운드 소스를 확보하는 작업이 필요하다.

### 우선 고려할 기준

- 무료 여부
- 상업적 이용 가능 여부
- 저작권/라이선스가 명확한지 여부
- attribution(출처 표기) 요구 여부

### 권장되는 확보 방식

1. 무료 음원 사이트 활용
   - Pixabay Sound Effects
   - Mixkit
   - Freesound
   - Incompetech
   - OpenGameArt

2. 라이선스 우선 순위
   - 가장 안전한 선택: `CC0`, `Public Domain`, 또는 명시적으로 “commercial use allowed”로 표기된 파일
   - 다음으로 고려: attribution이 필요한 파일
   - 단, 라이선스가 불명확한 파일은 사용하지 않는 것이 좋다.

3. 필요 시 자체 제작
   - 아주 짧은 비프음이나 간단한 알림음은 Web Audio API로 직접 생성할 수도 있다.
   - 이 경우 저작권 문제가 거의 없다.

---

## 6. 결론

현재 요구사항을 기준으로 보면, 알림음 재생 방식은 HTML5 `Audio` 객체가 가장 적합하다.

특히 여러 알림이 거의 동시에 발생하는 상황을 고려하면, “이전 재생을 중단하고 새 사운드를 즉시 재생”하는 제어 방식이 가장 자연스럽다.

또한 오디오 파일은 아직 없으므로, 이후 구현 전에 무료이고 상업적 이용 가능성까지 확인된 음원 소스를 확보하는 과정이 선행되어야 한다.
