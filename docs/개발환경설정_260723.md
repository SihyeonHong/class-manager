# 프론트엔드 개발 환경 및 도구 통합 가이드

## 도구 도입 목적 및 기대 효과

프로젝트에 코드 품질 및 포맷 관리 패키지를 도입함으로써, 코드를 작성하고 저장하는 순간부터 프로젝트의 공통 규칙이 일관되게 자동 적용되도록 만드는 것이 목표입니다.

- 형식 규칙 일원화: e.g., 공백과 줄바꿈, 사용하지 않는 코드, import 구조와 절대경로 별칭, Tailwind CSS 클래스 순서
- 자동화: 저장 시점에 자동으로 코드를 정리하도록 하여 개발 편의 향상.
- 환경 독립적 적용: 개인의 전역 VS Code 설정이나 운영체제(Windows, macOS 등)에 구애받지 않고, 프로젝트 내에서 동일한 편집 경험과 결과를 제공

## 설치한 패키지 및 환경

위 목표를 달성하기 위해 개발 의존성으로 다음 도구들을 설치했습니다.

- 'prettier': 프로젝트의 공통 코드 포맷터
- 'prettier-plugin-tailwindcss': Tailwind CSS 클래스 자동 정렬
- 'eslint-config-prettier': ESLint와 Prettier의 포맷 관련 규칙 충돌 방지
- 'eslint-plugin-import-x': import 구조와 경로 검사
- 'eslint-import-resolver-typescript': TypeScript 설정과 경로 별칭을 이용한 import 해석
- 'eslint-plugin-unused-imports': 사용하지 않는 import 탐지 및 자동 제거
- 'typescript-eslint': TypeScript 코드에 ESLint를 적용하기 위한 도구 모음

설치 시점(2026.07.23)의 'package.json'과 lockfile에 기록된 버전을 기준으로, 현재 프로젝트의 ESLint 9, Next.js 16, TypeScript 5 환경에 맞춰 설정이 구성되었습니다.

## ESLint import 검사 규칙 (eslint.config.mjs)

'eslint.config.mjs' 파일은 기존 Next.js ESLint 설정 위에 import 관련 flat config를 추가합니다. ESLint 9의 flat config 형식을 사용하여 여러 설정을 배열로 배치하고, 위에서 아래 순서로 필요한 규칙을 조합합니다.

- 'nextVitals'와 'nextTs': Next.js의 Core Web Vitals 및 TypeScript 권장 규칙을 적용합니다.
- 'importX.flatConfigs.recommended' / 'typescript': import 문법과 TypeScript import를 검사하는 기본 규칙을 추가합니다.
- 'files': JavaScript, JSX, TypeScript, TSX 등의 소스 파일에만 설정을 적용합니다.
- 'plugins': 사용하지 않는 import를 검사할 플러그인을 등록합니다.
- 'settings': TypeScript resolver와 Node resolver를 사용해 경로를 해석하며, 이를 통해 'tsconfig.json'의 '@/*' 별칭도 인식합니다.

규칙 값은 'off', 'warn', 'error'로 지정되며, 사용하지 않는 import는 강제로 정리되도록 'error'로 설정했습니다. 일반적인 미사용 변수는 'warn'으로 설정하되, '_'로 시작하는 변수와 인자는 의도적 미사용으로 간주하여 즉각적인 작업 중단 없이 정리 대상을 확인하게 합니다.

ESLint의 주요 검사 항목은 다음과 같습니다.
- import가 파일의 올바른 위치에 존재하는지 ('import-x/first')
- 동일한 모듈을 중복해서 불러오지 않는지 ('import-x/no-duplicates')
- import 경로가 실제로 해석되는지 ('import-x/no-unresolved')
- 외부 패키지와 내부 import가 일관된 순서로 배치되는지 ('import-x/order')

특히 'import-x/order'는 import를 builtin, external, internal, 상대 경로 그룹으로 나누고 그룹 사이 빈 줄을 강제하며, 내부 '@/**' 경로는 internal로 묶어 대소문자 구분 없이 이름순으로 자동 정렬합니다.

## Prettier 포맷 적용 (prettier.config.mjs)

프로젝트 공통 포맷 규칙을 'prettier.config.mjs'에 정의하여 CLI 명령만으로 동일한 결과를 보장합니다. ESM을 사용하는 환경에서 타입 힌트와 플러그인을 명확하게 다루기 위해 '.prettierrc' 대신 이 파일을 채택했습니다.

- 'printWidth: 100': 한 줄 길이의 줄바꿈 판단 기준입니다.
- 'semi: true': 문장 끝에 세미콜론을 강제합니다.
- 'singleQuote: false': 문자열에 큰따옴표를 사용합니다.
- 'trailingComma: "all"': 다중 줄 객체, 배열, 매개변수의 마지막 항목 뒤에 후행 쉼표를 추가합니다.
- 'arrowParens: "always"': 단일 인자 화살표 함수에도 항상 괄호를 씌웁니다.

### 줄바꿈 통일 (LF)
'endOfLine: "lf"' 옵션을 통해 파일의 줄바꿈 문자를 Unix 계열의 LF로 강제합니다. 이는 Windows 사용자의 CRLF 저장으로 인해 Git diff상 전체가 변경된 것으로 오인되는 현상을 막고, Linux 기반 CI/CD 환경과의 호환성을 맞추기 위함입니다.

### 들여쓰기 공백 처리
- 'tabWidth: 2': 들여쓰기 한 단계를 2칸 폭으로 설정합니다.
- 'useTabs: false': 실제 탭 문자 대신 공백(Space)을 사용합니다.
결과적으로 "들여쓰기 1단계마다 공백 2개를 사용"하게 되어, 편집기 설정에 따라 화면 정렬이 틀어지는 것을 방지합니다.

## Tailwind 클래스 자동 정렬
